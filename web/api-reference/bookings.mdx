## Module location

- **Routes**: `encuadrado_web/modules/bookings/routes.py`
- **Services**: `encuadrado_web/modules/bookings/services/`
- **Model**: `encuadrado_web/models/booking_model.py`

## Overview

The bookings module is the core scheduling engine. It handles appointment creation, confirmation, cancellation, rescheduling, and status management.

## Key models

| Model | Description |
|-------|-------------|
| `BookingModel` | Primary booking entity with status, datetime, service, patient |
| `TemporalBookingModel` | Temporary booking state during the creation flow |

## Booking lifecycle

```
Created → Confirmed → Completed
    │         │
    ▼         ▼
 Cancelled  No-show
```

| Module | Purpose |

## Background tasks

Booking-related tasks run on the `booking_notifications` queue:
- Confirmation emails and SMS
- Reminder notifications
- No-show follow-ups
- Calendar sync updates
