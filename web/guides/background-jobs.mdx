## Overview

Background jobs run on **Celery** with a **Redis** broker. Tasks are defined in `modules/[feature]/tasks.py` and use the `@shared_task` decorator with a dedicated queue.

## Task queues

| Queue | Purpose |
|-------|---------|
| `celery` | Default queue |
| `tracker` | Analytics and tracking events |
| `booking_notifications` | Booking confirmation and reminder notifications |
| `push_notifications` | Mobile push notifications |
| `email_notifications` | Email sending |
| `whatsapp_notifications` | WhatsApp messages |
| `invoice_issuing` | Invoice generation |
| `invoice_health_insurance_reimbursement_requests` | Insurance claim processing |
| `fonasa_bonds` | FONASA bond processing |
| `fonasa_bonds_monthly_invoices` | Monthly FONASA invoice generation |
| `rnpi` | Professional verification (Registro Nacional) |
| `reconciliator` | Payment reconciliation |
| `transbank_secondary_commerce_creation` | Transbank merchant setup |
| `update_sii_session_cookies` | SII (tax authority) session management |
| `logs` | Log processing |
| `fintoc_mx_inbound_transfer_matching` | Fintoc MX inbound transfer matching |

## Writing tasks

```python
from celery import shared_task

@shared_task(queue='booking_notifications')
def send_booking_confirmation(booking_id):
    booking = BookingModel.query.get(booking_id)
    if not booking:
        return

    # Send confirmation logic...
```

Place tasks in the module's `tasks.py` file and assign them to the appropriate queue.

## Task patterns

### Idempotent tasks

Design tasks to be safe to retry:

```python
@shared_task(queue='email_notifications', max_retries=3)
def send_email(email_id):
    email = EmailModel.query.get(email_id)
    if not email or email.sent:
        return  # Already sent or doesn't exist

    # Send and mark as sent
    deliver_email(email)
    email.sent = True
    db.session.commit()
```

### Chaining tasks

For multi-step workflows, call tasks sequentially from the first:

```python
@shared_task(queue='booking_notifications')
def process_booking_notifications(booking_id):
    send_booking_confirmation.delay(booking_id)
    send_provider_notification.delay(booking_id)
```
