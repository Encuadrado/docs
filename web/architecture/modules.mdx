## Structure

Each feature lives in `encuadrado_web/modules/[feature]/` with a consistent structure:

```
modules/[feature]/

‚îú‚îÄ‚îÄ routes.py         # Flask Blueprint with API endpoints
‚îú‚îÄ‚îÄ services/         # Business logic (or services.py for small modules)

‚îÇ   ‚îî‚îÄ‚îÄ [domain].py
‚îú‚îÄ‚îÄ serializer.py     # Data transformation for API responses
‚îú‚îÄ‚îÄ tasks.py          # Celery background tasks
‚îú‚îÄ‚îÄ middlewares.py    # Route decorators
‚îú‚îÄ‚îÄ helpers.py        # Module-specific utilities
‚îî‚îÄ‚îÄ signals.py        # Event handlers (optional)
```

<Note>
Use a single `services.py` file for small modules. Switch to a `services/` directory when the logic grows complex enough to warrant splitting.
</Note>

## Components

### Routes

Each module exposes a Flask Blueprint registered in the app factory:

```python
from flask import Blueprint, jsonify, request
from .serializer import UserSerializer
from .services.subscription import SubscriptionService

api = Blueprint('user_api', __name__)

@api.route('/users/<int:user_id>', methods=['GET'])
def get_user(user_id):
    user = UserModel.get_by_id(user_id)
    return jsonify(UserSerializer.one(user))
```

### Services

Business logic lives in service classes with static methods:

```python
class SubscriptionService:
    @staticmethod
    def get_active_subscription(user_id: int) -> Optional[Subscription]:
        return Subscription.query.filter_by(
            user_id=user_id,
            status='active'
        ).first()

    @staticmethod
    def create_subscription(user_id: int, plan_id: int) -> Subscription:
        subscription = Subscription(user_id=user_id, plan_id=plan_id, status='active')
        db.session.add(subscription)
        db.session.commit()
        return subscription
```

### Serializers

Transform model instances into API response dictionaries:

```python
class UserSerializer:
    @staticmethod
    def one(user: UserModel, only=None, include=None) -> dict:
        schema = UserSchema(only=only)
        schema.context = {'include': include, 'instance': user}
        return schema.dump(user)

    @staticmethod
    def many(users, only=None, include=None) -> List[dict]:
        schema = UserSchema(only=only, many=True)
        schema.context = {'include': include, 'instances': users}
        return schema.dump(users)
```

### Middlewares

Route decorators for cross-cutting concerns. Common decorators from `encuadrado_web/modules/middlewares.py`:

| Decorator | Purpose |
|-----------|---------|
| `@requires_auth_2` | Session authentication |
| `@requires_valid_user` | Validates user and injects `user` parameter |
| `@requires_existing_booking` | Validates booking exists |
| `@validates_query_fields` | Query parameter validation |

### Tasks

Celery background tasks with dedicated queues:

```python
from celery import shared_task

@shared_task(queue='booking_notifications')
def send_booking_confirmation(booking_id):
    booking = BookingModel.query.get(booking_id)
    # Send notification...
```
