## Prerequisites

- Node.js 19+
- Redis running locally
- A `.env` file with the required variables (ask the team for a template)

## Setup

<Steps>
  <Step title="Install dependencies">
    ```bash
    npm install
    ```
  </Step>
  <Step title="Set up environment variables">
    Create a `.env` file in the project root. See `.env.example` for reference. At minimum you need:

    ```bash
    REDIS_URL=localhost:6379
    ```
  </Step>
  <Step title="Build">
    ```bash
    npm run build
    ```
  </Step>
  <Step title="Start the worker (Terminal 1)">
    The worker must start before the app.

    ```bash
    npm run dev:worker
    ```
  </Step>
  <Step title="Start the app (Terminal 2)">
    ```bash
    npm run dev
    ```
  </Step>
</Steps>

<Info>
  The worker is long-lived — only restart it when worker code changes. The app is stateless and can be restarted freely.
</Info>

## Available scripts

```bash
npm run dev:worker    # Start WhatsApp worker (start this first)
npm run dev           # Start Express app with hot reload
npm run build         # Compile TypeScript to dist/
npm start             # Start production app
npm run start:worker  # Start production worker
./deploy.sh           # Pull, build, and restart app via PM2 (does not touch the worker)
```

## Add a WhatsApp client

Once both processes are running, register a new WhatsApp session:

```bash
curl -X POST http://localhost:3000/clients \
  -H "Content-Type: application/json" \
  -H "encuadrado-signature: <secret>" \
  -d '{"clientId": "your-client-id"}'
```

The worker will generate a QR code. Scan it with WhatsApp to authenticate the session.
