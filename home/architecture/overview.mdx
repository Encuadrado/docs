## Base path

The app is mounted at `/cuenta` via `next.config.mjs`. All routes are relative to this base path (e.g., `/cuenta/calendario`, `/cuenta/pacientes`).

## Provider stack

The root layout wraps the app in nested providers:

```
ThemeProvider
  └── QueryProvider (TanStack Query, 60s stale time, 1 retry)
      └── FlaskSessionProvider (auth from backend)
          └── SentryProvider
              └── LocaleProvider (i18n)
                  └── CenterProvider (multi-center context)
                      └── SidebarProvider
                          └── HeaderProvider
```

## Data flow

```
Flask Backend API (port 5000)
    │
    ▼
Axios instance (lib/utils.ts, with credentials)
    │
    ▼
TanStack Query (caching, refetching)
    │
    ▼
React components (via useQuery hooks)
```

All API calls go through the Axios instance configured in `lib/utils.ts`, which sets `withCredentials: true` and custom headers. TanStack Query handles caching with a 60-second stale time.

## Key patterns

### Feature-based pages

Each feature under `app/` is self-contained with its own components, state, and data fetching. Pages use `'use client'` directives for client-side rendering.

### Context API for shared state

Domain-specific contexts (Flask session, center member, sidebar state) are exposed via custom hooks in `hooks/`:

| Hook | Purpose |
|------|---------|
| `useFlaskSessionContext` | Current user session from backend |
| `useCenterMemberContext` | Active center and member selection |
| `useMobile` | Responsive breakpoint detection |
| `useToast` | Toast notifications |

### UI components

Base components in `components/ui/` wrap Radix UI primitives with Tailwind styling. Feature components compose these into domain-specific UI.

### Internationalization

Uses `react-i18next` with 11 Spanish locale variants (`messages/es-CL.json` is the primary). Chilean Spanish is the default locale.
