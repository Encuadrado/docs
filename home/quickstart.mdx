---
title: "Quickstart"
description: "Set up the Encuadrado Home development environment."
---

<Info>
**Prerequisites**: Node.js 22.x and pnpm.
</Info>

<Steps>
<Step title="Clone the repository">

```bash
git clone https://github.com/Encuadrado/encuadrado-professional-home.git
cd encuadrado-professional-home
```

</Step>

<Step title="Configure environment variables">

```bash
cp .env.example .env.local
```

Edit `.env.local`:

```env
NODE_ENV=development
NEXT_PUBLIC_API_BASE_URL=http://localhost:5000
NEXT_PUBLIC_AUTH_URL=http://localhost:5000/cuenta
NEXT_PUBLIC_SESSION_URL=http://localhost:5000/api/v2/users/get-logged-user-data
NEXT_PUBLIC_SENTRY_DSN=your_sentry_dsn_here
```

<Note>
The Flask backend (Encuadrado Web) must be running on port 5000 for the app to work.
</Note>

</Step>

<Step title="Install dependencies and start">

```bash
pnpm install
pnpm dev
```

The app runs at `http://localhost:3000/cuenta`.

</Step>
</Steps>

## Common commands

| Command | Description |
|---------|-------------|
| `pnpm dev` | Dev server with hot reload |
| `pnpm build` | Production build |
| `pnpm test` | Run Vitest in watch mode |
| `pnpm test:run` | Single test run |
| `pnpm test:ui` | Vitest UI mode |
| `pnpm lint` | Run Biome linter |
| `pnpm lint:fix` | Auto-fix lint issues |
| `pnpm format` | Format with Biome |
| `pnpm fix` | Format + lint:fix combined |
| `pnpm typecheck` | TypeScript type checking |
| `pnpm storybook` | Local Storybook (deprecated — use [storybook.encuadrado.com](https://storybook.encuadrado.com/)) |

## Troubleshooting

<AccordionGroup>
  <Accordion title="Node.js version mismatch">
    This project requires Node 22. Use nvm to switch:

    ```bash
    nvm install 22
    nvm use 22
    ```
  </Accordion>

  <Accordion title="Port 3000 already in use">
    ```bash
    pnpm dev -- -p 3001
    ```
  </Accordion>

  <Accordion title="Build errors after dependency changes">
    ```bash
    pnpm store prune
    rm -rf node_modules
    pnpm install
    ```
  </Accordion>

  <Accordion title="Google Fonts network issues during build">
    This is a network connectivity issue. Dev mode (`pnpm dev`) works offline. For production builds, ensure access to `fonts.googleapis.com`.
  </Accordion>

  <Accordion title="Sentry auth token warnings">
    Expected in development. For production, set `SENTRY_AUTH_TOKEN` in your environment.
  </Accordion>
</AccordionGroup>
